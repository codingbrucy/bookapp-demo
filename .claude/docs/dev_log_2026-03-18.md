# Dev Log — 2026-03-18

## Feature: Interactive Room Cards + Extracted BedGrid

### What Changed
Room cards on the dashboard are now **clickable** — they expand to show a detailed bed-by-bed view with cartoonized person figure SVGs for occupied beds and empty bed outlines for available slots. A hover highlight (teal border glow + lift) signals interactivity. The inline `BedGrid` was extracted to a shared component.

---

## New Components

### `components/PersonFigure.tsx`
Reusable cartoonized person silhouette SVG. Uses `currentColor` for color control via Tailwind `text-*` classes. **This is a placeholder** — the user will provide custom person SVGs to swap in later. The component accepts a `className` prop for sizing.

```
Props: { className?: string }  (default: "w-6 h-10")
```

### `components/BedGrid.tsx`
Extracted from the inline function in `page.tsx`. Now supports two render modes:

| Mode | When Used | What It Shows |
|---|---|---|
| `compact` | Collapsed room card (dashboard default) | Small teal squares — filled for occupied, dashed outline for empty |
| `detailed` | Expanded room card (on click) | Larger tiles in a 2-col grid with PersonFigure SVG, bed number, and patient name |

```
Props: { beds: Bed[]; mode?: "compact" | "detailed" }
```

**Key change**: BedGrid now takes the full `Bed[]` array instead of just `total`/`occupied` counts. This enables per-bed rendering (names, individual SVGs) in detailed mode.

### `components/RoomCard.tsx`
Encapsulates the entire room card with expand/collapse + hover behavior.

**States:**
- **Collapsed (default)**: compact BedGrid + capacity bar + patient name list. Identical to the previous static card.
- **Hovered**: see **Hover Polish** section below
- **Expanded (on click)**: teal ring (`ring-2 ring-[#0D7377]/10`) + detailed BedGrid with person SVGs + capacity bar

**Animation**: Uses the CSS `grid-template-rows: 0fr → 1fr` technique for smooth expand/collapse without Framer Motion. A chevron rotates to indicate state.

**Interaction**: Click anywhere on the card to toggle. `cursor-pointer` + `select-none` on the wrapper.

### Hover Polish (added same day)

Every detail on the room card transitions on hover — the goal is a premium, tactile feel without being noisy:

| Element | At Rest | On Hover |
|---|---|---|
| **Card** | `border-[#ede9e2]`, flat | lifts `-translate-y-1`, teal-tinted shadow, `border-[#0D7377]/30` |
| **Room number** | solid `#1a1a1a` | **color fill**: letters progressively turn amber (`#F4A261`) from left to right over 0.5s, stay amber while hovered (see `.claude/docs/css-text-fill-effect.md`) |
| **Wing label** | `text-[#9ca3af]` | transitions to `text-[#0D7377]/50` |
| **Left accent bar** | invisible (`scale-y-0`) | teal gradient bar slides down from top (`scale-y-100`), stays visible when expanded |
| **Background** | white | ultra-subtle teal gradient wash from left edge (`from-[#0D7377]/[0.03]`) |
| **Status badge** | emerald-50 bg | deepens to emerald-100 |
| **Chevron** | `text-[#9ca3af]` | transitions to `text-[#0D7377]` |
| **Patient initials** | `bg-[#0D7377]/10` | deepens to `bg-[#0D7377]/15` |
| **Patient names** | `text-[#6b7280]` | darkens to `text-[#374151]` |

**Room number color fill** (`globals.css`):
- At rest: `color: #1a1a1a` — completely plain text, no gradient, no `background-clip`
- On `group:hover`: a two-tone gradient (amber left, dark right) at `200%` width is clipped to text. Animation slides `background-position` from `100%` (dark visible) to `0%` (amber visible) over 0.5s with `ease-out`. `forwards` keeps the amber end state.
- On un-hover: all hover-only properties are removed — text snaps back to `color: #1a1a1a`
- Full educational breakdown in `.claude/docs/css-text-fill-effect.md`

**Teal-tinted shadows** (`globals.css`):
- `.room-card-glow`: base transition for `box-shadow` + `transform`
- `group:hover .room-card-glow`: `rgba(13,115,119,0.10)` outer + `rgba(13,115,119,0.06)` inner — a brand-colored depth effect instead of generic gray
- `.room-card-glow-active`: stronger version for expanded cards

## Design Decisions

**Why expand-in-place vs. modal or route?**
An expandable card keeps context — you can see the expanded room alongside other rooms and the pending intakes sidebar. A modal would obscure the dashboard. A route (/rooms/[id]) would be overkill for a quick bed check. The accordion pattern feels natural for a dashboard.

**Why CSS grid animation instead of Framer Motion?**
Framer Motion isn't installed yet. The `grid-template-rows: 0fr → 1fr` trick gives a smooth, native height animation without any library. When Framer Motion is added later (planned for SVG floor plan), the animation can optionally be upgraded.

**Why PersonFigure uses `currentColor`?**
This makes it trivially recolorable via Tailwind `text-*` classes. Occupied beds use `text-[#0D7377]` (teal). If we later want to color-code by program type or acuity level, it's a one-class change.
