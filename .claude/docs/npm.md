# npm — What It Is, How We Use It, and Common Commands

## What is npm?

**npm** (Node Package Manager) is the default package manager for Node.js. It does three things:

1. **Installs packages** — downloads third-party code (libraries, frameworks, tools) your project depends on
2. **Manages dependencies** — tracks exactly which packages and versions your project needs via `package.json`
3. **Runs scripts** — lets you define and run shorthand commands (like `npm run dev`) instead of typing long terminal commands

When you run `npm install`, npm reads `package.json`, fetches all listed dependencies from the npm registry (a giant public repository of open-source packages), and saves them into a `node_modules/` folder locally.

---

## Key Files

| File | Purpose |
|---|---|
| `package.json` | Lists your project's dependencies and scripts. You edit this. |
| `package-lock.json` | Locks exact versions of every installed package (including sub-dependencies). Do **not** edit manually — npm manages it. Commit this to git. |
| `node_modules/` | Where all installed packages live locally. **Never commit this** — it's regenerated by `npm install`. |

---

## Dependencies vs. devDependencies

In `package.json`, packages are split into two groups:

- **`dependencies`** — needed at runtime (in production). Example: `next`, `react`
- **`devDependencies`** — only needed during development/build. Example: `typescript`, `tailwindcss`, `eslint`

When you install a package:
- `npm install <package>` → adds to `dependencies`
- `npm install <package> --save-dev` → adds to `devDependencies`

## Common Commands

All commands below should be run from the `facility_panel/` directory:
```bash
cd facility_panel
```

### Daily Development
```bash
npm run dev          # Start the local dev server → localhost:3000 (hot reloads on save)
npm run build        # Compile the app for production (run before deploying)
npm run start        # Serve the production build locally (after npm run build)
npm run lint         # Check code for lint errors
```

### Package Management
```bash
npm install                        # Install all dependencies listed in package.json
npm install <package>              # Install a new package and add to dependencies
npm install <package> --save-dev   # Install a new package and add to devDependencies
npm uninstall <package>            # Remove a package
npm update                         # Update all packages to latest allowed versions
```

### Useful Info Commands
```bash
npm list             # Show all installed packages (flat)
npm list --depth=0   # Show only top-level packages (cleaner view)
npm outdated         # Show which packages have newer versions available
npm -v               # Show your npm version
node -v              # Show your Node.js version
```

```
You write code (TypeScript + JSX + Tailwind)
        ↓
npm run dev
        ↓
Next.js dev server starts (port 3000)
        ↓
Tailwind scans your files and generates CSS on the fly
        ↓
TypeScript compiles your code and catches type errors
        ↓
Browser hot-reloads when you save
```

When the Laravel backend is added later, npm stays on the **frontend side only**. Laravel uses **Composer** (PHP's equivalent of npm) for its own dependency management.

## Quick Tips

- If you pull down code changes from git and something breaks, run `npm install` first — a new dependency may have been added.
- Never delete `package-lock.json` manually — it's what makes installs reproducible across machines.
- `node_modules/` can be safely deleted and regenerated anytime with `npm install`. It's often hundreds of MB.
